# Desktop build (Tauri v2)

This folder wraps the Leather Pattern Designer as a native Windows `.exe` using
[Tauri v2](https://tauri.app). It is **fully isolated** from the app itself:

- The canonical app stays `../index.html` — open it in any browser, no build needed.
  That is always the fallback. **Nothing in this folder modifies it.**
- The desktop build wraps a *copy*: `build.rs` copies `../index.html` into
  `dist/index.html` at build time (the root file is only ever read). Delete this
  whole `desktop/` folder and the app is completely unaffected.

So if a Tauri/Rust change ever breaks, you lose the `.exe`, not the app.

## One-time setup

Tauri needs the Rust toolchain (Node is **not** required — the app is a single
static file). WebView2 (the runtime Tauri uses on Windows) is already installed.

Run these in a terminal (the `winget` step may prompt for confirmation):

```powershell
# 1. Install Rust
winget install Rustlang.Rustup --accept-source-agreements --accept-package-agreements
rustup default stable

# 2. Install the Tauri v2 CLI (compiles from source; takes a few minutes)
cargo install tauri-cli --version "^2.0.0" --locked

# 3. Generate the app icon set from the placeholder (or your own square PNG, >=512px)
cd "C:\Users\souls\.local\bin\Leather Stuff\desktop\src-tauri"
cargo tauri icon ..\icon-source.png
```

> Step 3 must run before the first compile — Tauri embeds the icon at build time.

## Daily use

From `desktop/src-tauri`:

```powershell
cargo tauri dev      # live desktop window (rebuilds on change)
cargo tauri build    # produce the distributable
```

`cargo tauri build` outputs:

- the raw executable -> `src-tauri/target/release/leather-pattern-designer.exe`
- an installer       -> `src-tauri/target/release/bundle/nsis/*-setup.exe`

The frontend copy refreshes automatically on every build (via `build.rs`), so you
only ever edit `../index.html`.

## Known integration point (not yet wired)

The app's save/load uses the browser **File System Access API**
(`showSaveFilePicker`). WebView2 may not expose it, in which case the app falls
back to its download path. The proper desktop fix is to route save/load through
Tauri's `dialog` + `fs` plugins. Tracked in `../MD files/CONTEXT.md` backlog. Until then,
verify save/load behaviour in the `cargo tauri dev` window.

## Layout

```
desktop/
  README.md            this file
  icon-source.png      placeholder logo (replace with a real one anytime)
  dist/index.html      build-time copy of ../index.html (git-ignored, auto-synced)
  src-tauri/
    Cargo.toml         Rust deps (tauri, tauri-build)
    build.rs           syncs ../index.html -> dist/ + runs tauri-build
    tauri.conf.json    window size, bundle targets, frontendDist -> ../dist
    src/main.rs        ~6-line host process
    capabilities/      Tauri v2 permission set for the main window
    icons/             generated by `cargo tauri icon` (step 3)
```
